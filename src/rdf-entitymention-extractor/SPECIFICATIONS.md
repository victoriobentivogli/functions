# RDF EntityMention Extractor (rdf-entitymention-extractor)

## Description

This function will receive RDF strings and extract relevant entity mentions (and sub-entity mentions) for the Entity Resolution Engine (ERE).

## Input

- `rdf_string`: RDF string
- `entity_type`: Entity type to extract
- `origin_id`: Unique identifier for the data origin
- `request_id`: Client-assigned request identifier
- `property_names`: List of property names to extract

## Output

- `entity_mention`: An `EntityMention` model instance (from the `ere-model` package), or `None` on error / no match.

The `EntityMention` model is populated with the following fields at this pipeline stage:

```python
EntityMention(
    origin_id="<origin_id>",
    request_id="<request_id>",
    entity_type="<entity_type>",
    entity_id="<auto-generated SHA256>",
    original_asset="<entity_uri>",
    properties={
        "<property_name>": "<property_value>"
    },
    # ML fields left at defaults — filled by downstream pipeline stages
    embedding=[],
    cluster_id="",
    similarity_score=0.0,
    confidence_score=0.0,
    is_medoid=False,
)
```

All the declarations that are reference to other elements will be traversed and collected into one level.

## Error Handling

In case of an error, return `None` and print the error message in the console.
