# RDF EntityMention Extractor

Extracts typed entity mentions from RDF strings and returns them as validated `EntityMention` model instances (Pydantic), suitable for ingestion into the **Entity Resolution Engine (ERE)**.

---

## Requirements

| Tool | Version |
|------|---------|
| Python | >= 3.12 |
| [uv](https://docs.astral.sh/uv/) | any recent |
| `make` | any recent |

---

## Installation

```bash
# From the src/rdf-entitymention-extractor directory
uv sync
```

To include development dependencies (pytest):

```bash
uv sync --extra dev
```

## Usage

```python
from rdf_entitymention_extractor import rdf_entitymention_extractor

rdf_string = """
@prefix rdf:   <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
@prefix org:   <http://www.w3.org/ns/org/> .
@prefix epo:   <http://data.europa.eu/a4g/ontology#> .
@prefix cccev: <http://data.europa.eu/m8g/> .
@prefix adms:  <http://www.w3.org/ns/adms#> .
@prefix skos:  <http://www.w3.org/2004/02/skos/core#> .
@prefix locn:  <http://www.w3.org/ns/locn#> .
@prefix xsd:   <http://www.w3.org/2001/XMLSchema#> .

<http://data.europa.eu/a4g/resource/notice-001/Organization/ORG-0001>
    a org:Organization ;
    epo:hasLegalName "FORSYNING HELSINGØR A/S"@da ;
    epo:hasBuyerLegalType <http://publications.europa.eu/resource/authority/buyer-legal-type/ra> ;
    epo:hasPrimaryContactPoint <http://data.europa.eu/a4g/resource/notice-001/ContactPoint/CP1> ;
    cccev:registeredAddress <http://data.europa.eu/a4g/resource/notice-001/Address/ADDR1> ;
    adms:identifier <http://data.europa.eu/a4g/resource/notice-001/Identifier/ORG-0001> .

<http://data.europa.eu/a4g/resource/notice-001/ContactPoint/CP1>
    a cccev:ContactPoint ;
    epo:hasContactName "Charlotte Nordberg" ;
    cccev:email "cnor@servia.dk" ;
    cccev:telephone "4576962100" .

<http://data.europa.eu/a4g/resource/notice-001/Address/ADDR1>
    a locn:Address ;
    locn:fullAddress "Energivej 25, Helsingør, 3000, DNK" ;
    locn:postCode "3000" ;
    locn:postName "Helsingør" ;
    epo:hasCountryCode <http://publications.europa.eu/resource/authority/country/DNK> .

<http://data.europa.eu/a4g/resource/notice-001/Identifier/ORG-0001>
    a adms:Identifier ;
    epo:hasScheme "organization" ;
    skos:notation "ORG-0001" .
"""

entity_type = "http://www.w3.org/ns/org/Organization"
property_names = [
    "hasLegalName",
    "hasPrimaryContactPoint",
    "registeredAddress",
    "identifier",
]
mention = rdf_entitymention_extractor(
    rdf_string,
    entity_type,
    origin_id="notice-001",
    request_id="req-550e8400",
    property_names=property_names,
)
print(mention)
```

> **Note:** Properties like `hasBuyerLegalType` are excluded because they are not
> listed in `property_names`. Pass `property_names=None` (or omit the argument) to
> retrieve all properties.
>
> Use `mention.model_dump()` to convert to a plain dict, or `mention.model_dump_json()` for JSON.

## API

### `rdf_entitymention_extractor(rdf_string, entity_type, origin_id, request_id, property_names) -> EntityMention | None`

| Parameter | Type | Description |
|-----------|------|-------------|
| `rdf_string`      | `str`       | RDF document (RDF/XML, Turtle, N-Triples, N3, or JSON-LD) |
| `entity_type`     | `str`       | URI of the `rdf:type` to extract |
| `origin_id`       | `str`       | Unique identifier for the data origin, used for tracking |
| `request_id`      | `str`       | Client-assigned request identifier, used for tracking |
| `property_names`  | `list[str]` or `None` | Optional. Top-level predicate local-names to include. Sub-keys (dot-notation) for each name are also kept. When `None` (default), all properties are returned. |

**Returns** an `EntityMention` instance (from the `ere-model` package) with the identification, original asset, and properties fields populated. ML fields (`embedding`, `cluster_id`, etc.) are left at their defaults for downstream pipeline stages.

Returns `None` if no entity of the given type is found or on error.

#### Reference flattening

When a property value is a URI that itself has triples in the graph, its sub-properties are merged into the parent entity's `properties` dict using **dot-notation** keys.

For example, if an `org:Organization` has `epo:hasPrimaryContactPoint` pointing to a `cccev:ContactPoint` node that carries `cccev:email "cnor@servia.dk"`, the organisation's properties will contain:

```
{
    "hasPrimaryContactPoint.email": "cnor@servia.dk",
    ...
}
```

This traversal is recursive (e.g. `registeredAddress.hasCountryCode`) and **cycle-safe**.

#### Multi-valued properties

When the same predicate appears more than once for the same subject, the property value is stored as a **list** of strings.

#### Error handling

On any parsing or extraction error, the function returns `None` and prints a descriptive message to stdout prefixed with `[rdf-entitymention-extractor]`.

---

## Supported RDF formats

The parser attempts the following formats in order:

1. RDF/XML
2. Turtle
3. N-Triples
4. N3
5. JSON-LD
